# AI 素材自动化测试

## 能力说明

| 能力 | 需求 | 触发方式 | 技术层级 |
|------|------|---------|---------|
| AI 素材分层与测试分配 | 1.3 | 手动 + 被 creative-lifecycle 调用 | L4 |

通过 AI 视觉分类对大批量素材做智能分层，自动分配到不同测试路径，缩短从素材产出到效果验证的时间。

## 触发条件

1. **手动触发**: 飞书指令 "AI 分层测试这批素材"
2. **被 creative-lifecycle 调用**: 新素材上传后可自动触发 AI 分层

## 执行步骤

### 一、AI 素材分层与测试分配（需求 1.3）

**输入**: `project_id`, `asset_ids[]`, `test_template_id`（可选）, `budget_cap`

**步骤**:

1. **提交 AI 标签任务**:
   提交 AI 标签任务，获取任务 ID 和状态

2. **等待标签结果**（超时 10 分钟）:
   查询素材 AI 标签结果，获取标签列表（含置信度）、主要风格、钩子类型和 CTA 类型

3. **智能分层**: 按标签分类素材:

   | 标签分类 | 素材特征 | 测试路径 |
   |---------|---------|---------|
   | 强剧情 | `primary_style` 含 "剧情/故事/情感" | "吸量 + 付费"双目标测试 |
   | 强福利 | `primary_style` 含 "福利/优惠/活动" | "吸量 + 付费"双目标测试 |
   | 强玩法 | `primary_style` 含 "玩法/策略/养成" | "纯付费"测试 |
   | 其他 | 不属于以上分类 | 默认测试路径 |

   **低置信度处理**: if 标签 `confidence < 0.6` → 标记为"待人工判定"

4. **测试预算分配**:
   ```
   total_budget <= budget_cap
   强剧情/强福利类: 各分配 35% 预算
   强玩法类: 分配 20% 预算
   其他: 分配 10% 预算
   ```

5. **推送分层方案到飞书**（等待人工确认）:
   ```
   🤖 AI 素材分层方案 — 项目XX

   共 N 条素材，分为 3 个测试组:

   📌 强剧情组（M 条）— 吸量+付费测试
   - 素材A（置信度: 92%）
   - 素材B（置信度: 85%）
   预算: $X

   📌 强玩法组（K 条）— 纯付费测试
   - 素材C（置信度: 88%）
   预算: $Y

   ⚠️ 待人工判定（J 条）:
   - 素材D（置信度: 55%，AI 判为"强剧情"但不确定）

   总预算: $Z（上限: $<budget_cap>）

   请确认分层方案后，将自动创建测试广告。
   ```

6. **人工确认后执行**:
   - 创建测试批次，设置素材列表、测试模板、AI 分配模式和预算上限
   - 调用 creative-lifecycle Skill 搭建测试广告

7. **记录到 memory**:
   ```
   [AI 测试] 批次 X: N 条素材分 3 组
   - 强剧情 M 条 → 吸量+付费测试, 预算 $A
   - 强玩法 K 条 → 纯付费测试, 预算 $B
   - 待人工判定 J 条
   ```

## 输入/输出

### 输入参数

| 参数 | 类型 | 说明 | 必填 |
|------|------|------|------|
| `project_id` | string | 项目 ID | 是 |
| `asset_ids[]` | string[] | 待分层的素材 ID 列表 | 是 |
| `test_template_id` | string | 测试模板 | 否（默认项目模板） |
| `budget_cap` | number | 测试预算上限 | 是 |

### 输出

| 场景 | 推送方式 | 内容 |
|------|---------|------|
| 分层方案 | 飞书消息 | 各分类素材列表 + 测试路径 + 预算 |
| 确认后 | 飞书通知 | 测试广告创建完成 |

## 安全规则

1. **预算上限**: 单批测试总预算不超过 `budget_cap`
2. **人工复核**: AI 分层方案**必须经人工确认后才能执行**
3. **低置信度标记**: confidence < 0.6 的分类需标记为"待人工判定"
4. **先记录再行动**: 所有操作前先记录到 memory
