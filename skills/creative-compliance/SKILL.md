# 素材合规性预检

## 能力说明

| 能力 | 需求 | 触发方式 | 技术层级 |
|------|------|---------|---------|
| 素材合规性检查 | 1.5 | 被 creative-lifecycle 调用 + 手动 | L4 |

在素材上线前调用 AI 模型服务进行文本和视觉合规评分，输出风险等级和修改建议，降低拒审和封禁风险。

## 触发条件

1. **被 creative-lifecycle 调用**: 素材上传流程中的合规预检环节
2. **手动触发**: 飞书指令 "检查素材合规性"、"合规预检 asset_id=XXX"

## 执行步骤

### 一、素材合规性检查（需求 1.5）

**输入**: `asset_id`, `project_id`, `channel_targets[]`, `locale`

**步骤**:

1. **提交合规检测任务**:
   提交素材合规检测任务，获取任务 ID 和预估完成时间

2. **轮询等待结果**（超时 5 分钟）:
   查询合规检测结果，获取风险等级、得分、违规项和修改建议

3. **解析检测结果**:

   **文本层面**（BERT 文本分类）:
   - 检测违规词: 绝对化承诺（"最好"、"第一"）、敏感词汇、误导性表述
   - 对照各渠道政策规则（Meta 政策 / TikTok 政策 / Google 政策）

   **视觉层面**（视频/图像内容模型）:
   - 检测敏感画面: 暴力、色情、侵权内容
   - 检测违规 UI 元素: 模拟系统通知、虚假关闭按钮

4. **判定风险等级**:
   - `risk_level == "high"` → **拦截素材上线**
     - 推送飞书消息: 拦截通知 + 违规项列表 + 修改建议
     - 返回给 creative-lifecycle: `action = "block"`
   - `risk_level == "medium"` → **标记风险，建议人工复核**
     - 推送飞书消息: 风险提醒 + 违规项 + 建议人工确认
     - 返回给 creative-lifecycle: `action = "review"`
   - `risk_level == "low"` → **通过**
     - 返回给 creative-lifecycle: `action = "pass"`

5. **输出合规检查报告**:
   ```
   🛡️ 合规检查报告 — 素材 "<asset_name>"

   风险等级: 🔴高风险 / 🟡中风险 / 🟢低风险
   综合评分: X/100

   违规项:
   1. [文本] "限时最低价" → 绝对化承诺（严重）
      建议: 改为"限时优惠价"
   2. [视觉] 第3秒画面 → 模拟系统通知 UI（中等）
      建议: 移除或替换为品牌内 UI

   渠道政策匹配:
   - Meta: ❌ 不符合（项目1违反广告政策3.1）
   - TikTok: ⚠️ 需注意（项目2可能被审核标记）
   - Google: ✅ 符合
   ```

6. **记录到 memory**: `[合规检查] 素材"XX" → 风险等级: high/medium/low, 违规项 N 条`

## 输入/输出

### 输入参数

| 参数 | 类型 | 说明 | 必填 |
|------|------|------|------|
| `asset_id` | string | 素材 ID（DAP 素材库） | 是 |
| `project_id` | string | 项目 ID | 是 |
| `channel_targets[]` | string[] | 目标投放渠道 | 是 |
| `locale` | string | 目标语言/地区 | 否（默认 en-US） |

### 输出

| 场景 | 返回值 | 飞书推送 |
|------|--------|---------|
| 高风险 | `action = "block"` | 拦截通知 + 修改建议 |
| 中风险 | `action = "review"` | 风险提醒 + 人工复核请求 |
| 低风险 | `action = "pass"` | 无（静默通过） |

## 安全规则

1. 高风险素材**必须被拦截**，不可跳过上线
2. 合规检查结果需**留存**，支持后续审计
3. 模型评分仅作为**辅助**，中风险需人工最终判定
4. 不修改素材本身，只输出评估结果
